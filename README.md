## 職務概要

_2024年3月_ に法政大学理工学部を卒業後、株式会社千葉銀行にAI/MLエンジニアとして入行。約1年半にわたり、生成AI（LLM）と画像認識を活用した業務効率化プロジェクトを、企画立案からモデル開発、アプリケーション運用まで一貫して主導。具体的には、

### 1. 本部の企業サポート部向け　融資決裁権限に関する問い合わせアシスタントAIの開発
- 期間：2024年11月~2025年3月（5カ月）で、QAアシスタントアプリケーションの設計から開発、運用を担当(エンジニア2名）
- 対象：企業サポート部の5名の問い合わせ担当者
- 課題：月80件に及ぶ融資決裁権限に関する問い合わせ対応負荷の軽減とニュアンスなど回答品質の標準化

#### 背景概要

　銀行では、融資実行の可否を決定する**決裁権限**に関するマニュアルが存在し、融資案件が支店長等の裁量内に該当するか、もしくは本部への正式な稟議が必要かを記載されている。融資には細心の注意が必要な上、マニュアルを読んでも解決しないケースが多く、稟議が必要な案件かどうかを問い合わせチャットボットで受け付けている(月80件)。合わせて、業務側から、マニュアルには記載されていない細かな部分や回答のニュアンスなどのpreferenceを維持したい要望を頂戴した。以上のことから、問い合わせ対応の負荷と回答品質の標準化を目的に、AIを活用して、マニュアルそのものではなく、PIIをマスキングした過去のQ&Aペアを用いたRAGによるQ&Aアシスタントソリューションを開発、運用。

#### 役割、アクション
- チャットボット内の、過去5000ペアのQAセット内の個人情報のマスキング作業、csvの項目整形(項目、質問、回答)、回答列のフォーマット化(回答、条件、参照)
- ChromaDBによるベクトルデータベースの作成、langchainを用いたRAGシステムの構築
- pywidgetを利用したアプリケーションUIの構築
- Human in the loopの観点でUI上での評価システムの実装
  
#### 成果
- 担当者による評価で、Accuracy**80%**の達成
- 従前のマニュアルの手動参照と比較した検索時間の短縮、ニュアンスを維持したことによる、**過去回答の言葉遣いが参考になる**点などが評価され、問い合わせ対応工数の削減に貢献

#### 技術スキル
RAG, langchain, ChromaDB, ipywidgets

### 2. 市場業務部向け　国際収支項目番号検索AI(外国送金マニュアルのRAGシステム)

- 期間： 2024年11月～2025年3月（5カ月）で、RAGアプリケーションの設計から開発、運用を担当(エンジニア2名）
- 対象：市場業務部の行員8名程度
- 課題：日本銀行発行の80ページ長のマニュアルを都度参照する工数、および本部への業務集中に伴う負荷軽減

#### 背景概要

　市場業務部では、送金の目的や理由に応じて指定されている3桁の番号である**国際収支項目番号**を日本銀行への報告する必要がある外国送金業務を取り扱っている。そこで、日本銀行から発行された80ページものマニュアルを参照しており、多大な時間を消費している。また、中期経営計画でも挙げている**意思決定や情報の本部集中**に伴い、業務量の増加や新人へのナレッジシェアが課題となってくる。以上のことから、マニュアルを参照する工数の削減及び、新任でも過去のケースやマニュアルなどの該当箇所を参照できるRAGソリューションを開発、運用。

#### 役割、アクション
- 業務課題のヒアリングから要件定義、開発、アプリケーションのデプロイまでをE2Eで担当
- PDFのチャンク化、ベクトルDBの作成、Langchain/ChromaDBを用いたRAGシステムの構築
- メタデータを活用した元PDF参照ページの表示機能をstreamlitのUI上に実装
- コンテナアプリケーションとして、Google CloudのCloud Runでデプロイ・運用

#### 成果
- 担当者による評価で、Accuracy**80%**の達成(correct/helpful/incorrectで評価)
- マニュアル参照時間の削減

#### 技術スキル
Cloud Run, GCS, Streamlit, RAG, langchain, ChromaDB

### 3. デジタル戦略部業務改革G向け　提案制度振り分けAI(改善提案の重複判定・担当部署自動割り当てといった、提案掲示板運営の効率化を図るAIエージェントの開発)
- 期間：2025年3月～2025年5月（3カ月）でアプリケーションの設計から開発、運用を担当(エンジニア1名)
- 対象：デジタル戦略部業務改革Gの提案制度システム受付担当者(月担当2名)
- 課題：新規提案の重複判定と担当部署の割り当てプロセスの効率化(15分程度)

#### 背景概要

　デジタル戦略部業務改革Gでは、支店の行員から日々の業務で感じた業務の改善案や提案を受け付けている掲示板を４年ほど運用しており、過去4000件のエントリーがある。担当者は、新しく来た提案に対して過去に重複する提案がないか、合わせて新提案の場合に、どの部署が対応すべきかの割り振り作業に1件あたり15分程の時間を費やしている。このプロセスの効率化のため、AIエージェントによるソリューションを構築。

#### 役割、アクション
- 課題のヒアリング
- Langgraphを活用したAIエージェント(重複判定エージェントと所管部振り分けエージェント)の設計・開発
- 過去4000件のデータ内の個人情報マスキングとデータセットの整形処理を実施
-  Langchain/ChromaDBを用いたRAGシステムの構築
-  Streamlitを活用したUIを構築し、Google CloudのCloud Runにデプロイ・運用
- choromaベクトルデータベースの作成
- GCSや一時データ、Streamlitをうまく連携させ、担当者が新しい提案をUI上でリアルタイムで更新できる仕組みを実現(データベース更新の負荷を軽減)

#### 成果
- 提案処理プロセスの効率化(1件あたり平均15分から約1分)
- 類似検索システムの観点で新規スタッフからのポジティブな評価
- 年間約1,000件の提案処理に対し、**約233時間の業務工数削減**（14分/件 × 1,000件）

#### 技術スキル
- Cloud Run, GCS, Artifact Registry, Streamlit, RAG, langchain, langgraph, ChromaDB

### 4. 広告出稿主向け　デジタルサイネージの広告効果測定AI(視聴人数、属性(世代や性別)、交通人数)の開発・実証実験
- 期間： 2024年11月～現在 で、アプリケーションの設計から開発、運用を担当(エンジニア1名)
- 対象：デジタル戦略部広告チーム、既存・新規のデジタルサイネージ広告主
- 課題：2026年の大型デジタルサイネージの拡大を見据えた、広告出稿主に対する出稿結果(視聴人数、属性、交通人数)のデータ還元やマーケティング活用。また他行との差別化など。

#### 背景概要

 弊社では、デジタルサイネージ広告業を取り扱っているが、広告出稿主に対して結果の還元が出来ていない(マス業界全体を見ても同様の課題あり)。2026年大型デジタルサイネージの拡大や他行との差別化の観点で、効果測定結果のデータを還元をすることで、既存のお客様の継続や新規の獲得を狙うため、AIによる統計データの還元ソリューションを開発する。フェーズを２段階に分けており、フェーズ1では、支店前の交通人数を測定のAIの開発とPoC、フェーズ2では、サイネージの試聴判定や属性分析といった統計情報も測定をするAIソリューションの開発とPoCに分けてプロジェクトに着手。現在は両モデルの開発や検証や展開に向けた個人情報の整理・リスクの評価を実施済みで、PoCをしつつ本展開に向けて改善点を洗い出している。

#### 役割とアクション
- GCP上で、MediapipeやDeepSORTによるモデルの構築・検証(フェーズ0)
- USBカメラとローカルPC上でフェーズ1,2のモデルの構築、検証
- コンプライアンス・リスク統括部と連携し、検証や展開に向けた個人情報保護観点での弁護士相談や利用目的の策定等
- カメラ解像度、モデルサイズ、PCやエッジデバイスの性能を考慮した、精度と速度のトレードオフ検証および要件定義
- 他社ヒアリング

#### 成果
- ローカル環境でのPoCにより技術的な実現性を検証。今後の展開に向けた開発の方向性、カメラ性能、プライバシー保護戦略を明確化
- 新しいビジネス展開に向けた技術的・戦略的基盤を確立

#### 技術スキル
YOLO, mediapipe, ultralytics, PINTO_model, centroid-base-tracking, DeeoSORT, nvidia-jetson, ...
