### 1. 本部の企業サポート部向け　融資決裁権限に関する問い合わせアシスタントAIの開発
- **期間**：2024年11月~2025年3月（5カ月）
- **体制**：エンジニア2名(QAアプリケーションの設計・開発・運用を担当）
- **対象**：企業サポート部の問い合わせ担当者5名
- **ビジネス課題**：月80件に及ぶ融資決裁権限に関する問い合わせ対応の負荷軽減と回答品質の標準化

#### 背景概要
　銀行業務における融資実行の可否を決定する**決裁権限マニュアルは極めて複雑であり、融資案件が支店長等の裁量内に該当するか、もしくは本部への正式な稟議が必要かを詳述している。融資には細心の注意が必要な上、こちらのマニュアルを読んでも解決しないケースが多いため、本部担当者への問い合わせがチャットボットを通じて頻発していた。また、現場からはマニュアルに記載のない細かな判断基準や回答のニュアンス(Preference)を維持したいという強い要望があった。

#### 役割、アクション
- **データ前処理と匿名化**：過去5,000ペアのQAセットから個人情報（PII）をマスキングし、CSV形式（項目、質問、回答）へ整形。回答列を「回答・条件・参照」に構造化し、LLMが解釈しやすいフォーマットを定義。
- **RAG基盤の構築**: ChromaDBを用いたベクトルデータベースおよびLangChainによるRAGシステムを設計。
- **Human-in-the-Loop(HITL)の実装**: AIの回答をそのまま出力せず、人間が検証・修正を行うプロセスを確立。評価や責任の分離のためだけでなく、「新規の質問に対するAI回答と、それに対する人間の最終回答」をペアで保管・蓄積するデータパイプラインで運用中。
- **継続的学習サイクルの確立**: 蓄積された正解データを用いて定期的にベクトルデータベースを更新し、検索精度と回答品質を継続的に向上させるスキームを実装。
- **UI開発**: ipywidgetsを用いたアプリケーションUIの構築。
  
#### 成果
　担当者評価でAccuracy80%を達成。検索時間の短縮に加え、熟練者の言葉遣いを参考にした回答が可能になり、ナレッジの平準化と工数削減に貢献。


#### Technical Stack
RAG, LangChain, ChromaDB, ipywidgets

### 1. S社_BigQueryを用いたブランダマスタ紐付けの検証PoC(RAG/BQML)
- 期間：2025年12月~2026年1月（2カ月, エンジニア1名、PM1名）
- 対象：S社(大手飲料メーカー）
- 課題：開発・運用コストの削減、コスト、精度等を含めた実現可能性の検証

#### 背景概要

　顧客先では、Google Cloud上で本番運用システムを幾つか運用しているが、Cloud FunctionやWorkflowなどを用いることによるインフラの複雑性や開発・運用コストを懸念いている。BQMLといったSQLコマンドでAI機能やスケジュール実行などのBig Query上の機能を用いることで、RAGを用いた**商品名とブランド名の紐付けタスクの実現可能性を検証**するため着手。

#### 役割、アクション
- GCSからBig Queryへのデータのロード
- 商品名の略称に対応するための拡張データの作成
- 重複削除のためのUDF関数の作成(NFCK、文字列置換、ユニーク化）
- AI処理前の完全一致結合
- Remote model機能を用いて、埋め込みモデル(gemini-embeddig-001)で商品名とブランド名をベクトル化したデーブルの作成
- ベクトル検索で算出された、コサイン距離の近い上位５件をプロンプトに組み込ませ、テキスト生成モデル(gemini-flash-2.5)でブランド名を判断理由とともにJSON形式で推定
- プロシージャとして上記のSQLをワークフローとして定義し、定期実行スケジュールの作成
- 1526件の精度検証可能な商品名を用いた検証(1500件の商品データで、前処理による商品名の80%削減、正解率80%、　実行時間1分半、料金数十円)
  
#### 成果
- 運用コスト削減：Big QueryのみでRAGを実現し、インフラ管理不要化による運用工数の削減
- LLM処理コストの削減：商品名の前処理で80%削減、また完全一致結合を事前に行うことで、LLMへの入力トークンコストを削減
- 拡張知識の利用：略称問題を解決し、商品名などのニュアンスが複雑なケーズでの自動紐付けの実現性を検証
- 処理速度・料金：BQMLによるLLM機能ではデフォルトでバッチ処理が有効なため、LLMの処理コストが半減し、かつ処理速度の向上が見込める

#### 技術スキル
SQL, BQML, Google Cloud, Big Query
